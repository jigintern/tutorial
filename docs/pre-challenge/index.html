Deno について学習

今回のインターンでは Deno を使った Web アプリを作成します。

事前準備として以下の資料を参考に Deno について学習し、簡単な Web アプリを作成して提出してください。

Deno とは？

JavaScript / TypeScript の実行環境です。

JavaScript を使ってバックエンドのサーバを作ることができます。

フロントエンドばかりでサーバのプログラムを書いたことのない方も、フロントエンドと同じ JavaScript で書くことができるので簡単に習得できます。

JavaScript でサーバというと Node.js が挙げられますが、作者が Node.js の反省から Deno を作ったので、より便利なものと思ってもらって大丈夫です！

サーバでのデータの保存（データベースの利用）

Deno Deploy ではプログラムからファイルの保存ができないので、サーバでデータを保存するには他の手段が必要になります。

Supabase というサービスを使って Postgres のデータベースを作成し、そこにデータを保存するようなプログラムを作ってみます。

まず、Supabase にデータベースを作成します。

https://app.supabase.io/

「Sign In With Github」をクリックします。

「Authorize supabase」をクリックします。

「New Project」をクリックします。

作成するデータベースの情報を入力します。

Name：データベースの名前です

Database Password：接続用のパスワードです。必要になるので忘れないように

Region：データベースを作成する地域です。どこでも大丈夫だと思います

入力したら「Create new project」をクリックします。

プロジェクトの作成が完了するまでしばらく待ちます。

完成しました。

左側の「Database」タブを選択、Setting の「Connection Pooling」を選択します。

設定画面の一番下にある「Connection string」にある postgres:// で始まる URL をコピーします。

これが作成したデータベースに接続する URL です。

[YOUR-PASSWORD] となっている部分は、先ほど入力したパスワードに置き換えて使います。

次に Deno のプログラムから、このデータベースに接続するようにします。

https://dash.deno.com/playground/tutorial-postgres

上記の URL のプロジェクトを開いて、画面上部中央の「Fork & Edit」をクリックします。

編集できるようになりましたが、データベースの接続 URL が設定されていないのでエラーになってしまいます。

画面上部左のプロジェクト名をクリックすると、プロジェクトの設定画面に移動できます。

Settings の Environment Variables から環境変数としてデータベースの URL を設定します。

KEY：DATABASE_URL（プログラムで指定している変数名）

VALUE：さきほど作成したデータベースの postgres:// で始まる URL

[YOUR-PASSWORD] は作成時に入力したパスワードで置き換えてください

入力したら「Add」をクリックすると登録されます。

Overview の Open Playground からプログラムに戻ります。

エラーがなくなりプログラムが起動しました！

ローカル環境で Deno

ローカル環境で Deno を開発できるようにします。

まず、Deno をインストールしましょう。

https://deno.land/

Installation の項目にあるスクリプトを自分の環境に合わせて実行してください。

以下のスクリプトを使うと実行ファイルの PATH を通す必要がないので簡単です。

Windows（PowerShell）

env:DENO_INSTALL = "C:\Program Files\deno"

iwr https://deno.land/x/install/install.ps1 -useb | iex

Mac、Linux

curl -fsSL https://deno.land/x/install/install.sh | sudo DENO_INSTALL=/usr/local sh

インストール済みの方は、以下のコマンドを実行すると最新版に更新できます。

deno upgrade

インストールが完了したら、以下のコマンドを実行してみましょう。

deno run https://deno.land/std/examples/welcome.ts

Welcome to Deno 🦕

と表示されれば OK です。

さきほど GitHub のリポジトリにエクスポートしたプロジェクトを開いてみます。

リポジトリをローカル環境にクローンして、そのフォルダに移動して、以下のコマンドを実行します。

deno run main.tsx

エラーが表示されました。

これは Deno のセキュリティ機能による制限で、デフォルトではファイルやネットワークの機能を利用することができません。

コマンドのパラメータとして許可する機能を指定することで実行できます。

例えば、--allow-net でネットワークアクセスを許可、--allow-read でファイルシステムの読み取りアクセスを許可といった具合です。

ここでは、ネットワークアクセスが許可されていないエラーメッセージが表示されましたが、プログラムとしてはファイルシステムの読み込みも必要となるので、すべてのパーミッションを許可する -A を指定することにします。

deno run -A main.tsx

今度はエラーなく実行できたようです。

ブラウザで http://localhost:8000 にアクセスしてみます。

ローカル環境で動きました！

プログラムの終了は Ctrl + C です。

これで、ローカル環境で開発 → GitHub のリポジトリにプッシュ → Deno Deploy にデプロイの一連の流れが出来上がりました！
