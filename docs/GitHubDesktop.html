GitHub Desktop を使った開発の流れ

プロジェクトをクローンする

まず開発するプロジェクトを GitHub リポジトリからクローンします

ブラウザでクローンするプロジェクトを開いて、Code の Open with GitHub Desktop をクリックする

GitHub Desktop が起動してリポジトリのクローン確認のダイアログが表示されるので、保存フォルダを確認して Clone をクリックする

GitHub Desktop は git を操作するだけのソフトで、コードを書いたりは別のソフトを使う

Open in Visual Studio Code ボタンをクリックすると Visual Studio Code が起動する

（ここで開くソフトは設定で変更可能）

Visual Studio Code でプロジェクトが開いた

ブランチを作成

新しい機能を追加する時はブランチを作成します

main ブランチには直接コミットしないのが一般的なルールです

New Branch ボタンをクリックする

ブランチ名を入力して Create Branch ボタンをクリックする

追加する機能にあったわかりやすい名前を付けるといいです

（この例の bbs は悪い例です）

手元のコードに変更がある場合は、こんなダイアログが表示されることがあります

Bring my changes to xxx を選択すれば OK です

これで新しく作ったブランチに切り替わりました

ファイルを編集してコミット

Visual Studio Code などエディタでファイルを編集します

編集があると、GitHub Desktop の方の Changes に編集したファイルと変更内容が表示されるようになります

編集した内容をコミットしていきます

コミットするファイルにチェックを付け、タイトルと説明文を入力し、Commit to xxx ボタンをクリックします

他の人が見てわかりやすいようにコミットを分けると親切です

この例だとサーバ側のプログラム server.js と、クライアント側のプログラム index.html を別々のコミットに分けました

こっちはクライアント側のコミットです

History の方でコミットの履歴が確認できます

手元のコミットをプッシュする

この時点ではコミットが自分の手元にあるだけの状態です

チームの他の開発者にコードの変更を反映するにはプッシュする必要があります

Publish origin ボタンをクリックします

（Push と Publish は言葉の違いだけなので気にせず）

これで GitHub リポジトリに手元と同じブランチが作成されました

プルリクエストを作成する

GitHub リポジトリに Push はしたけど、main ブランチには入っていない状態です

main ブランチに変更をマージするためにプルリクエストを作成します

Create Pull Request ボタンをクリックすると、ブラウザが起動します

プルリクエストのタイトルと説明を入力して Create pull request ボタンをクリックします

プルリクエストが作成されました

このプルリクエストをチームの他の人にレビューしてもらいます

Slack と連携しておくとプルリクエストが作成された時点で通知されるので捗ります

レビューして main ブランチにマージして問題なさそうなら Merge pull request ボタンをクリックします

確認されるので Confirm merge ボタンをクリックします

マージされました！

マージされたブランチは不要なので Delete branch しましょう

フェッチ＆プル

次の機能を開発する時には、最新の main ブランチの内容に更新しましょう

ブランチを main に戻します

Fetch origin ボタンをクリックします

これは GitHub リポジトリに更新があるか確認する操作で、まだ手元のファイルは更新されません

Pull origin ボタンをクリックします

これで手元のファイルが最新の内容に更新された

また新しいブランチを作成して次の機能の開発しましょう！
